## 1. 프로젝트 배경 및 목적

VeriPass는 **지하철·버스 개찰구에서 연령을 자동 인식해 부정승차를 탐지**하는 시스템이다.
카메라 앞을 지나가는 사람의 얼굴을 보고 **청소년(13–18세) / 성인(19–64세) / 고령자(65세 이상)** 세 구간 중 어디에 해당하는지 실시간으로 분류한다.

이 보고서는 해당 분류 과제에 쓸 수 있는 AI 모델 7개를 동일한 조건에서 비교해, **Raspberry Pi 5(이하 RPi5) 실물 배포 기준**으로 어떤 모델이 가장 적합한지 결론을 내린다.

> **왜 RPi5인가?**
> 개찰구에는 고성능 서버가 아닌 소형 임베디드 컴퓨터가 들어간다. 아무리 정확해도 너무 느리면 실시간 개찰구에 쓸 수 없으므로, 추론 속도(시간)를 가장 중요한 기준으로 삼는다.

## 2. 실험 조건

| 항목 | 내용 |
|---|---|
| 데이터셋 | AAF + UTKFace Asian (아시아 얼굴 중심) |
| 분류 목표 | 3-클래스: teen_13_18 / adult_19_64 / senior_65plus |
| 평가 샘플 | RPi5 현장 실측: 50장 (rpi50) |
| 전처리 | 동일 전처리, train/test 분리 유지 (데이터 누출 없음) |
| 측정 환경 | Raspberry Pi 5, CPU 추론 (GPU 없음) |
| 측정 방식 | 이미지 1장당 평균 추론시간(mean_latency_ms) + FPS 실측 |

> **50장 테스트의 의미**: 개수가 적어 통계적 분산이 있지만, RPi5 실물 배포 속도를 직접 잰 유일한 측정값이다. 각 수치는 방향성 참고용이며, 최종 판단에는 1,885장 전체 데이터셋 결과와 병행해서 해석한다.

## 3. 평가 지표 해설 (용어 정리)

처음 보는 사람을 위해 각 지표가 무엇을 의미하는지 설명한다.

### 3-1. macro-F1 (핵심 지표 ★)

F1은 **정밀도(Precision)**와 **재현율(Recall)**의 조화평균이다.
- **정밀도(Precision)**: "청소년이라고 판단한 것 중 실제 청소년 비율" → 오탐(멀쩡한 성인을 청소년으로 잘못 잡는 것)을 줄이는 능력
- **재현율(Recall)**: "실제 청소년 중 몇 명을 탐지했나" → 미탐(진짜 청소년인데 성인으로 놓치는 것)을 줄이는 능력

**macro-F1**은 청소년·성인·고령자 세 클래스 각각의 F1을 구한 뒤 단순 평균한 값이다. 성인이 전체의 86%를 차지하므로, 성인만 잘 맞혀도 accuracy는 높아 보일 수 있다. macro-F1은 이 불균형을 보정해 **세 그룹을 균등하게 잘 맞히는 모델**에 높은 점수를 준다.

> 값이 1.0에 가까울수록 좋다. 0.5 이하는 사실상 무작위 수준.

### 3-2. 청소년 recall (핵심 지표 ★)

"실제 청소년 몇 명 중 몇 명을 잡았나"의 비율. 부정승차 탐지의 핵심이다.
- recall = 0.60 → 청소년 10명이 지나가면 6명만 탐지, 4명은 성인으로 통과시킴
- recall = 0.00 → 청소년을 단 한 명도 잡지 못함 (사실상 탐지 불가)

### 3-3. 고령자 recall (핵심 지표 ★)

"실제 고령자 몇 명 중 몇 명을 잡았나". 복지 할인 대상자 보호 측면에서 중요하다.
고령자를 성인으로 잘못 분류하면 정당한 할인 수혜자를 막을 수 있다.

### 3-4. 나이 MAE (Mean Absolute Error)

"예측 나이와 실제 나이의 평균 오차(년)". 낮을수록 정확하다.
예: MAE = 6.5 → 평균적으로 실제 나이에서 ±6.5세 오차로 예측.

### 3-5. 성별 정확도

얼굴을 보고 성별을 맞히는 정확도. 이 프로젝트에서는 보조 지표이며 모든 모델이 95% 이상으로 큰 차이가 없다.

### 3-6. RPi5 추론시간 (핵심 지표 ★ — 이 보고서에서 최우선)

이미지 1장을 처리하는 데 걸리는 시간(초). **낮을수록 좋다.**
개찰구 특성상 사람이 지나가는 순간 판정이 끝나야 하므로, 수 초가 걸리는 모델은 실배포에 적합하지 않다.

## 4. 7개 모델 소개

이 프로젝트는 3개 백본 계열에서 총 7개 모델을 비교했다.

| 약칭 | 정식 명칭 | 설명 |
|---|---|---|
| **CNN** | MobileNetV3-Large CNN | 경량 분류 전용 CNN. 얼굴 검출 없이 이미지 전체를 입력받아 분류. 가장 단순하고 빠름 |
| **IF-base** | InsightFace buffalo_l (기본) | 얼굴 인식 특화 라이브러리. 성별·나이를 추정하는 원본 모델을 그대로 사용 |
| **IF-FT** | InsightFace buffalo_l (파인튜닝) | IF-base 백본을 고정하고, 상단에 3-클래스 분류 MLP 헤드를 추가로 학습시킨 버전 |
| **MV1-224** | MiVOLO v1 (224px, 기본) | 나이·성별 추정 특화 트랜스포머 모델. 입력 해상도 224px |
| **MV1-FT** | MiVOLO v1 (224px, 파인튜닝) | MV1-224 백본을 고정하고 3-클래스 MLP 헤드를 추가 학습 |
| **MV2-FT** | MiVOLO v2 (384px, 파인튜닝) | MV2-384 백본을 고정하고 3-클래스 MLP 헤드를 추가 학습 |
| **MV2-384** | MiVOLO v2 (384px, 기본) | MiVOLO 최신 버전. 해상도 384px로 v1보다 높은 품질. 기본 모델 사용 |

> **"파인튜닝(FT)"이란?** 이미 대용량 데이터로 학습된 강력한 모델(백본)을 가져와, 우리 데이터셋의 3-클래스 분류에 맞게 마지막 분류 레이어만 추가로 훈련시키는 기법. 백본 자체는 건드리지 않으므로 성별·나이 MAE는 base와 동일하다.

## 6. 그래프 해석

### [그래프 00] 종합 3패널 — 전체 개요

`graphs/00_overall.png`

세 개의 패널(A·B·C)을 한 화면에 보여준다.

- **(A) 분류 핵심지표**: macro-F1, 청소년 recall, 고령자 recall을 나란히 비교. IF-FT가 macro-F1 1위, MV1-FT·MV2-FT가 청소년 recall 공동 1위임을 한눈에 볼 수 있다.
- **(B) 나이 MAE**: CNN이 6.51로 가장 낮고(정확), InsightFace 계열이 9.59로 가장 높다(부정확). MiVOLO 계열은 중간 수준.

### [그래프 01] macro-F1 — 분류 균형 종합 성능

`graphs/01_macro_f1.png`

**IF-FT(0.720)가 가장 높고, IF-base(0.439)가 가장 낮다.**

주목할 점:
- IF-base → IF-FT: 파인튜닝으로 0.439 → 0.720으로 크게 개선 (+0.281). InsightFace 원본 모델은 3-클래스 분류에 최적화되지 않았기 때문에, 헤드 추가 학습 효과가 뚜렷하다.
- MV2-384 base(0.501)는 그래프에서 IF-base 다음으로 낮게 나오는데, 이는 50장 소규모 테스트의 샘플 편차 영향이 크다. 전체 1,885장 테스트에서는 MV2-384가 macro-F1 0.782로 1위였다는 점을 함께 참고해야 한다.
- MV1 계열은 파인튜닝 후 오히려 하락(0.613 → 0.538). 청소년 recall을 높이면서 precision이 낮아진 트레이드오프 결과다.

### [그래프 02] 청소년 recall — 부정승차 탐지 핵심 지표

`graphs/02_teen_recall.png`

**MV1-FT·MV2-FT 공동 1위(0.800). IF-base가 0.000으로 꼴찌.**

- IF-base(0.000): 청소년 50장 중 단 한 명도 탐지 못함. 원본 InsightFace 모델의 나이 추정 범위와 우리 데이터 분포가 맞지 않아 모두 성인으로 분류하는 문제.
- CNN(0.200): 전체 5명 중 1명만 탐지. 속도는 빠르지만 부정승차 탐지에는 미달.
- MV2-384 base(0.400): 파인튜닝 없이는 청소년 탐지가 부족함을 보여준다.
- 파인튜닝(-FT) 모델들은 전반적으로 base보다 청소년 recall이 높다. 이는 학습 시 class-weighted 손실함수로 소수 클래스를 강조했기 때문이다.

### [그래프 03] 고령자 recall — 취약계층 보호 지표

`graphs/03_senior_recall.png`

**MV2-FT가 0.750으로 1위. MV1 계열 전체와 MV2-384 base, CNN이 0.250으로 공동 꼴찌.**

- IF 계열(base·FT 모두 0.667): InsightFace는 고령자 탐지에 비교적 강한 특성을 보인다.
- MV2-FT(0.750): 청소년과 고령자 recall을 동시에 끌어올린 유일한 모델. 다만 추론이 3.27초로 느리다는 치명적 단점이 있다.
- MV1 계열은 파인튜닝 후에도 고령자 recall이 개선되지 않음(0.250 유지). MV1 파인튜닝이 청소년 recall 향상에만 집중한 결과로 보인다.

### [그래프 05] 3-클래스 정확도 — 단순 정답률

`graphs/05_3class_accuracy.png`

**IF-FT가 0.870으로 1위. MV1-FT가 0.720으로 최하위.**

> ⚠️ 단순 정확도의 함정: 성인이 전체의 82%(50장 중 41장)를 차지하므로, 모든 입력을 "성인"으로 찍어도 82%가 나온다. 따라서 이 지표는 macro-F1보다 신뢰성이 낮다. macro-F1과 함께 해석해야 한다.

MV1-FT(0.720): 파인튜닝 후 청소년 recall은 올라갔지만 성인 일부를 청소년으로 잘못 분류하는 오탐이 늘어나 전체 정확도는 오히려 하락했다.

### [그래프 06] 나이 MAE — 연령 추정 정확도

`graphs/06_age_mae.png`

**CNN이 6.51로 1위(가장 정확). InsightFace 계열이 9.59로 가장 부정확.**

- CNN(6.51): 학습 손실함수에 나이 추정이 포함되어 있어 연속 나이 예측 성능이 좋다.
- InsightFace(9.59): 특히 teen_13_18 클래스의 나이 오차가 17.6세로 매우 크다. 청소년 얼굴을 30대 초반으로 추정하는 경향이 있어, base 모델의 나이 추정만으로는 청소년 분류가 불가능하다.
- MiVOLO 계열(7.51 / 7.92): 중간 수준. base와 FT가 동일한 이유는 FT는 3-클래스 헤드만 교체하고 백본(나이 추정 부분)은 그대로 두기 때문이다.

### [그래프 07] CNN 학습 곡선 — 모델 학습 과정 검증

`graphs/07_cnn_training_curve.png`

CNN 모델의 20 에폭 학습 과정을 보여준다.

- **train_loss(원형 실선)**: 에폭마다 꾸준히 감소 → 학습이 정상적으로 진행됨.
- **val_loss(사각형 실선)**: 약 에폭 7부터 0.95~1.05 구간에서 수렴. train_loss는 계속 내려가는데 val_loss가 거의 평탄해지므로 **약한 과적합(overfitting)** 징후가 있다.
- **val_MAE(점선, 우측 Y축)**: 초반 ~8.5년에서 에폭 20 기준 ~6.2년으로 감소. 중간에 진동이 있지만 전반적으로 하향 추세.

> **해석**: CNN 학습은 안정적으로 수렴했으나, val_loss가 더 이상 개선되지 않는 에폭 약 7–8 이후로는 조기 종료(early stopping)를 적용했어도 결과가 비슷했을 것이다. 20에폭 전체 학습이 과도하지는 않은 수준.

### [그래프 08] RPi5 추론시간 — 배포 가능성 핵심 지표

`graphs/08_rpi_inference.png`

**CNN이 0.1초로 압도적 1위. MV2 계열이 3.3초로 꼴찌.**

배포 관점에서 구간을 나누면:

| 구간 | 모델 | 실용성 |
|---|---|---|
| 0.1초 이하 | CNN | 개찰구 어디서나 즉시 가능 |
| 0.5–0.8초 | MV1-224, MV1-FT, IF-base, IF-FT | 개찰구 실시간 사용 가능 |
| 3.3초 이상 | MV2-FT, MV2-384 | 개찰구 실시간 배포 부적합 |

> ⚠️ 주의: MV1·MV2 수치는 백본 순수 추론시간이며, 실제 환경에서는 YOLO 얼굴 검출이 선행된다. YOLO 포함 시 MV1은 실제 ~3.8초, MV2는 ~6.5초 수준으로 증가한다. CNN과 InsightFace는 얼굴 검출 파이프라인이 통합되어 있어 위 수치가 전체 처리 시간에 가깝다.

### 8-3. 유의사항

1. **샘플 크기**: 이 비교는 50장 기준. 청소년은 5명, 고령자는 4명에 불과해 1명 차이가 recall 0.2 변동을 일으킨다. 최종 결정 전 더 많은 샘플로 검증 권장.

2. **YOLO 파이프라인 미포함**: MiVOLO 계열의 추론시간은 백본만 측정했다. 실제 배포 시 얼굴 검출 단계가 추가되면 MV1은 ~3.8초, MV2는 ~6.5초로 증가할 수 있다. InsightFace는 얼굴 검출이 파이프라인에 통합되어 0.8초 수치가 전체 처리 시간에 가깝다.

3. **파인튜닝 ≠ 무조건 향상**: MV1의 경우 파인튜닝 후 macro-F1이 오히려 하락(0.613 → 0.538)했다. 강한 base 모델에 파인튜닝을 적용하면 recall은 높아지지만 precision이 낮아지는 precision↔recall 트레이드오프가 발생한다. 운영 정책(미탐 우선 vs 오탐 우선)에 따라 base와 FT 중 선택이 달라진다.

4. **전체 테스트셋 결과와의 차이**: 1,885장 기준으로는 MV2-384 base가 종합 1위(macro-F1 0.782)였으나, RPi5 추론 속도(3.27초)가 배포 기준을 충족하지 못해 RPi 우선순위에서는 최하위로 내려갔다. 이 보고서의 순위는 "RPi5 배포 가능성"이 반영된 결과임을 명시한다.
